## Constitution Check

*GATE: Must pass before Phase 0 research. Re-check after Phase 1 design.*

- **Stack clarity**: Frontend stack is Streamlit. Any custom component or
  browser-side renderer integrated with Streamlit is justified as a dependency.
- **Performance**: Plan defines a reproducible 5,000-node gate for parse,
  layout, initial render, pan, and zoom with no UI freeze.
- **Dependency discipline**: Every new dependency is justified with purpose,
  performance/bundle impact, and a lighter alternative considered.
- **Testability**: Parsing, validation, graph model, layout, and render
  preparation are testable without mounting UI components.
- **Layer boundaries**: Parsing, graph model, layout, render preparation, and UI
  are separate layers with explicit contracts.
- **Input validation**: JSON node/edge input is validated against a schema before
  render, and user-facing error paths are planned.
